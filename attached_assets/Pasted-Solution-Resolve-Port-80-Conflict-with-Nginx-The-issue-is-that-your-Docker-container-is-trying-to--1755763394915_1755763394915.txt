Solution: Resolve Port 80 Conflict with Nginx

The issue is that your Docker container is trying to use port 80, but it's already occupied by Nginx. Here's how to fix this:
Step-by-Step Solution
1. Stop any existing AddyPin containers
bash

docker stop addypin-production 2>/dev/null || true
docker rm addypin-production 2>/dev/null || true

2. Run your Docker container on a different port (3000)
bash

docker run -d \
  -p 3000:5000 \
  --name addypin-production \
  --restart unless-stopped \
  -e NODE_ENV=production \
  -e DATABASE_URL="postgresql://addypin:your_db_password@localhost:5432/addypin" \
  addypin-app

3. Verify the container is running
bash

docker ps
sleep 3
curl http://localhost:3000/api/stats || echo "Container may still be starting"

4. Check container logs if needed
bash

docker logs addypin-production

5. Configure Nginx as a reverse proxy
bash

# Backup current nginx config
cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.backup

# Create AddyPin nginx configuration
cat > /etc/nginx/conf.d/addypin.conf << 'EOF'
server {
    listen 80;
    server_name addypin.com www.addypin.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
EOF

6. Test and reload Nginx
bash

nginx -t
systemctl reload nginx

7. Test the complete setup
bash

curl http://localhost/api/stats

Alternative Quick Fix

If you prefer to keep using port 80 directly with Docker and stop Nginx:
bash

# Stop Nginx temporarily
systemctl stop nginx

# Run Docker on port 80
docker run -d \
  -p 80:5000 \
  --name addypin-production \
  --restart unless-stopped \
  -e NODE_ENV=production \
  -e DATABASE_URL="postgresql://addypin:your_db_password@localhost:5432/addypin" \
  addypin-app

# Disable Nginx from starting at boot (optional)
# systemctl disable nginx

Verify Everything is Working

Check if your application is accessible:
bash

# Check if Docker container is running
docker ps

# Check application health
curl http://localhost:3000/api/stats

# Check if Nginx is properly proxying
curl http://localhost/api/stats

The reverse proxy approach (using Nginx to forward to port 3000) is generally recommended as it allows you to:

    Keep Nginx for SSL termination later

    Host multiple applications on the same server

    Benefit from Nginx's performance optimizations

    Easily add caching and other features

Your application should now be accessible at your domain name (addypin.com) through the Nginx reverse proxy.
