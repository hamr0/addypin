 CRITICAL ISSUE: Docker Disk Space Exhausted

The error is clear: "no space left on device" - your Docker environment has run out of disk space. This is a common issue when building large Node.js projects with many dependencies.
Immediate Solutions:
Option 1: Clean Docker System (Quick Fix)
bash

# Clean up Docker resources to free space
docker system prune -a --volumes --force

# Check available space
df -h

# Try building again with smaller base image
docker build -t addypin-app .

Option 2: Use a Smaller Base Image
bash

# Create a more optimized Dockerfile
cat > Dockerfile << 'EOF'
FROM node:20.19.3-alpine3.19

# Install build dependencies only
RUN apk add --no-cache \
    python3 \
    make \
    g++ \
    git

# Create non-root user
RUN addgroup -g 1001 -S runner && \
    adduser -u 1001 -S runner -G runner

# Set working directory
WORKDIR /app
RUN chown -R runner:runner /app

# Copy package files first
COPY --chown=runner:runner package*.json ./

# Switch to runner user
USER runner

# Install dependencies with clean cache
RUN npm ci --no-audit --prefer-offline && \
    npm cache clean --force

# Copy source code
COPY --chown=runner:runner . .

# Build the application
RUN npm run build

# Remove dev dependencies
RUN npm prune --omit=dev --no-audit

# Set production environment
ENV NODE_ENV=production

EXPOSE 5000
CMD ["npm", "start"]
EOF

Option 3: Build Outside Docker First
bash

# Build the application on your host system first
cd /opt/addypin/addypin-repo
npm ci
npm run build

# Then create a Dockerfile that only copies the built assets
cat > Dockerfile << 'EOF'
FROM node:20.19.3-alpine3.19

WORKDIR /app

# Copy only production files
COPY package*.json ./
COPY dist/ ./dist/
COPY server/ ./server/
COPY shared/ ./shared/

# Install only production dependencies
RUN npm ci --only=production --no-audit

EXPOSE 5000
CMD ["npm", "start"]
EOF

Option 4: Check and Expand Disk Space
bash

# Check current disk usage
df -h
du -sh /var/lib/docker/

# If using Docker Desktop, increase allocated space
# If on a VM, expand the disk size

Recommended Approach:

Start with Option 1 (cleanup) then Option 2 (Alpine base image). The Alpine version is much smaller and may solve the space issue immediately.

If that doesn't work, use Option 3 to build on your host system first, then create a minimal Docker image with just the built assets.

Run the cleanup first:
bash

docker system prune -a --volumes --force
df -h

Then try building with the Alpine-based Dockerfile. This should resolve the space issue.
