# Fixes Applied - October 5, 2025

## Issue 1: Staging API Validation Error ✅ FIXED

### Problem
- Frontend sends latitude/longitude as **numbers**
- Backend schema expected **strings**
- Result: 400 validation error on staging.addypin.com

### Solution
Updated `shared/schema.ts` to accept both types and convert to string:

```typescript
export const insertPinSchema = createInsertSchema(pins).omit({
  id: true,
  shortcode: true,
  createdAt: true,
  isActive: true,
}).extend({
  // Accept both string and number for latitude/longitude (convert to string)
  latitude: z.union([z.string(), z.number()]).transform(val => String(val)),
  longitude: z.union([z.string(), z.number()]).transform(val => String(val)),
});
```
